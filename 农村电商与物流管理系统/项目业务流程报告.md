# 农村电商与物流配送系统 - 完整业务流程报告

## 一、项目概述

### 1.1 项目定位
**农村电商与物流配送系统**是一个连接 **农户→电商平台→物流配送→消费者** 的一体化平台，旨在解决农产品上行难、物流配送效率低的问题，实现从田间地头到消费者餐桌的全链路数字化管理。

## 三、用户角色与权限体系

### 3.1 用户类型 (user_type)

| 类型值 | 角色 | 权限说明 |
|--------|------|----------|
| 1 | 消费者 | 浏览商品、下单购买、查看订单、管理收货地址、收藏商品 |
| 2 | 农户 | 店铺管理、商品发布/管理、订单处理、收入统计 |
| 3 | 物流人员 | 运单管理、配送路线、签收记录 |
| 4 | 管理员 | 系统全部权限，包括用户管理、商品审核、订单管理、数据统计 |

### 3.2 权限控制
- **JWT Token认证**：用户登录后获取Token，后续请求携带Token进行身份验证
- **路由守卫**：前端根据用户类型控制页面访问权限
- **接口权限**：后端通过Token解析用户身份，控制接口访问

## 四、完整业务流程

### 4.1 消费者购物流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  注册/  │───►│  浏览   │───►│  加入   │───►│  确认   │───►│  在线   │
│  登录   │    │  商品   │    │  购物车 │    │  订单   │    │  支付   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └────┬────┘
                                                                  │
                                                                  ▼
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  评价   │◄───│  确认   │◄───│  配送   │◄───│  农户   │◄───│  支付   │
│  晒单   │    │  收货   │    │  运输   │    │  发货   │    │  成功   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

#### 详细流程说明：

**1. 用户注册/登录**
- 消费者通过手机号/用户名注册账号
- 登录后获取JWT Token
- 可管理多个收货地址

**2. 商品浏览与搜索**
- 首页展示轮播图、推荐商品、分类导航
- 支持按分类、产地、销量、价格筛选
- 关键词搜索商品
- 查看商品详情（图片、价格、库存、农户信息）

**3. 购物车管理**
- 添加商品到购物车
- 修改商品数量
- 删除购物车商品
- 批量选择结算

**4. 订单创建**
- 选择收货地址
- 确认商品信息
- 填写订单备注
- 计算订单金额（商品总额 + 运费 - 优惠）

**5. 在线支付**
- 支付方式：微信支付、支付宝支付
- 支付成功后订单状态变为"待发货"
- 未支付订单15分钟后自动取消

**6. 订单跟踪**
- 待发货：等待农户发货
- 待收货：查看物流轨迹
- 已完成：确认收货后可评价
- 已取消：用户取消或超时未支付

**7. 评价晒单**
- 确认收货后可对商品进行评价
- 支持上传图片
- 评价后展示在商品详情页

### 4.2 农户经营流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  注册/  │───►│  店铺   │───►│  发布   │───►│  等待   │───►│  商品   │
│  登录   │    │  设置   │    │  商品   │    │  审核   │    │  上架   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └────┬────┘
                                                                  │
                                                                  ▼
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  收入   │◄───│  售后   │◄───│  确认   │◄───│  接收   │◄───│  接收   │
│  提现   │    │  处理   │    │  发货   │    │  订单   │    │  订单   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
```

#### 详细流程说明：

**1. 店铺设置**
- 设置店铺名称、头像、背景图
- 填写店铺描述、主营品类
- 设置店铺标签
- 管理发货地址

**2. 商品发布**
- 填写商品名称、描述
- 上传商品图片（主图+详情图）
- 设置价格、库存、产地
- 选择商品分类
- 提交审核

**3. 商品审核流程**
- 状态：待审核 → 审核通过/驳回
- 审核通过后自动上架
- 被驳回可修改后重新提交

**4. 订单处理**
- 接收新订单通知
- 查看订单详情
- 确认发货（填写物流信息）
- 处理退款/退货申请

**5. 收入统计**
- 今日/本月/累计收入
- 订单量统计
- 销售额趋势图表
- 收入提现申请

### 4.3 物流配送流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  登录   │───►│  接收   │───►│  打印   │───►│  上门   │───►│  运输   │
│  系统   │    │  运单   │    │  面单   │    │  取件   │    │  配送   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └────┬────┘
                                                                  │
                                                                  ▼
┌─────────┐    ┌─────────┐    ┌─────────┐
│  完成   │◄───│  确认   │◄───│  派送   │
│  配送   │    │  签收   │    │  送达   │
└─────────┘    └─────────┘    └─────────┘
```

#### 详细流程说明：

**1. 运单接收**
- 系统根据配送区域自动分配运单
- 物流人员查看待取件列表
- 批量接收运单

**2. 面单打印**
- 批量选择运单打印面单
- 支持电子面单
- 打印配送路线

**3. 上门取件**
- 按路线顺序上门取件
- 扫描确认取件
- 更新物流状态为"已揽件"

**4. 运输配送**
- 实时更新物流轨迹
- 系统推送轨迹更新
- 异常预警处理

**5. 签收确认**
- 送达后联系收件人
- 确认签收
- 更新订单状态为"已完成"

## 五、订单状态流转

```
                    ┌─────────────┐
                    │   待付款    │
                    │  (Status 0) │
                    └──────┬──────┘
                           │
           ┌───────────────┼───────────────┐
           │ 支付成功      │ 用户取消      │ 超时未支付
           ▼               ▼               ▼
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │   待发货    │ │   已取消    │ │   已取消    │
    │  (Status 1) │ │(CancelType1)│ │(CancelType1)│
    └──────┬──────┘ └─────────────┘ └─────────────┘
           │ 农户发货
           ▼
    ┌─────────────┐
    │   待收货    │
    │  (Status 2) │
    └──────┬──────┘
           │
     ┌─────┴─────┐
     │           │
     ▼           ▼
┌─────────┐ ┌─────────┐
│确认收货 │ │商家取消 │
└────┬────┘ └────┬────┘
     │           │
     ▼           ▼
┌─────────┐ ┌─────────┐
│ 已完成  │ │ 已取消  │
│(Status3)│ │(Cancel  │
└─────────┘ │ Type2)  │
            └─────────┘
```

### 订单状态说明

| 状态值 | 状态名称 | 说明 |
|--------|----------|------|
| 0 | 待付款 | 订单创建成功，等待用户支付 |
| 1 | 待发货 | 支付成功，等待农户发货 |
| 2 | 待收货 | 已发货，等待用户确认收货 |
| 3 | 已完成 | 用户确认收货，订单完成 |
| 4 | 已取消 | 订单被取消（用户取消/商家取消/超时） |

### 取消类型

| 类型值 | 类型名称 | 说明 |
|--------|----------|------|
| 0 | 未取消 | 订单正常进行 |
| 1 | 用户取消 | 用户主动取消订单 |
| 2 | 商家取消 | 商家/管理员取消订单 |

## 七、API接口设计

### 7.1 接口规范

- **基础路径**：`http://localhost:8080`
- **认证方式**：JWT Token (Header: Authorization: Bearer {token})
- **响应格式**：JSON
- **统一响应结构**：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {}
  }
  ```

## 十、总结

### 10.1 项目亮点
1. **完整的电商闭环**：从商品发布到订单完成的全流程覆盖
2. **多角色协同**：消费者、农户、物流人员、管理员各司其职
3. **实时物流跟踪**：完整的物流状态和轨迹跟踪
4. **数据可视化**：后台管理系统提供丰富的数据统计和图表
5. **权限控制完善**：基于JWT的多角色权限体系

### 10.2 业务流程核心
- **消费者**：浏览 → 下单 → 支付 → 收货 → 评价
- **农户**：开店 → 发布商品 → 处理订单 → 发货 → 收入提现
- **物流**：接单 → 取件 → 配送 → 签收
- **管理员**：审核 → 调度 → 监控 → 统计

### 10.3 技术实现特点
- 前后端分离架构
- RESTful API设计
- JWT身份认证
- MyBatis-Plus数据访问
- Element Plus UI组件库
- Pinia状态管理
